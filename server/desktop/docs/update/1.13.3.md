# 🚀 Cambios en el Sistema – Versión 1.13.3
**Área responsable:** Proyectos y Transformación Digital

---

## 🎯 Objetivo General

La versión **1.13.3** consolida el ciclo de desarrollo entre el 13 de marzo y el 17 de abril de 2026, abarcando la implementación completa del módulo de carnets de personal, la integración de la encuesta socioeconómica, el módulo de otros fletes con backend y frontend completos, contratos de personal, mejoras de rendimiento en el servidor, automatización del despliegue con GitHub Actions y múltiples correcciones de estabilidad.

Se priorizó la **trazabilidad de personal de planta** mediante carnets con código QR y serial SKU, la **captura de datos socioeconómicos** en el proceso de ingreso, la **gestión de fletes adicionales** con esquema propio en MongoDB y la **modernización del pipeline de despliegue**.

**Objetivos específicos:**
- Implementar generación de carnets en PDF con QR, vinilo, serial SKU y flujo de carnets definitivos.
- Integrar encuesta socioeconómica en el proceso de ingreso de personal.
- Implementar módulo de otros fletes: schema MongoDB, clase de negocio, endpoints API, WebSocket y UI completa en desktop.
- Agregar módulo de contratos de personal en gestión laboral.
- Optimizar rendimiento del servidor (latencia HTTP, workers, índices MongoDB).
- Automatizar el despliegue del servidor con GitHub Actions.
- Corregir errores detectados en producción en descartes, listas de empaque, volante calidad, carnets y formularios de limpieza.
- Restringir el cierre de contenedores hasta validar el ingreso completo a cuarto frío.
- Ampliar el modelo RBAC con accesos parciales a la colección de cargos.

---

## ⭐ Cambios Destacados

### 1. Módulo de Carnets de Personal

**Módulo Personal → Carnets**

Se implementó desde cero el flujo completo de generación y gestión de carnets de personal, tanto temporales como definitivos.

- **Generación de carnets en PDF con Worker Thread:** Los carnets se generan en segundo plano usando un worker thread dedicado, evitando bloquear el hilo principal del servidor.
- **Carnet definitivo con serial SKU:** Nuevo endpoint para crear carnets definitivos vinculando el serial SKU al registro del trabajador, con opción de vinilo.
- **Generación masiva de carnets:** Funcionalidad para generar carnets en lote con notificación en tiempo real al cliente vía WebSocket una vez completado el proceso.
- **Campo vinilo en personal:** Se agregó el campo `vinilo` al modelo de personal para distinguir carnets con acabado en vinilo.
- **Corrección de fotos en carnet:** Se resolvieron errores relacionados con el procesamiento y encabezado de imágenes al generar los PDFs.
- **Corrección de acumulación de listeners:** Se solucionó un bug de acumulación de listeners y rooms WebSocket durante la generación de carnets.
- **Evento `leave_job` en WebSocket:** Se agregó el evento para que el cliente pueda abandonar correctamente el job de generación de carnets.
- **Mejoras visuales en templates:** Ajustes de fuente, medidas y diseño del reverso del carnet.
- **Descuento de insumos por carnet:** Se implementó el método que descuenta el plástico del inventario de insumos al generar cada carnet.
- **Módulo carnets en desktop:** Integración completa en la app desktop con soporte para carnets finales, contador de plásticos usados y campo observaciones TF.

---

### 2. Encuesta Socioeconómica

**Módulo Personal → Ingreso de Personal**

- **Integración en ingreso de personal:** El formulario de ingreso ahora incluye la encuesta socioeconómica como parte del flujo de registro.
- **Refactor validación de cédula:** Se reestructuró la validación de cédula dentro del proceso de ingreso para mayor robustez.
- **Endpoint de encuestas pendientes:** Nuevo endpoint que consulta desde Redis las encuestas socioeconómicas pendientes, permitiendo identificar personal sin encuesta completada.

---

### 3. Módulo de Otros Fletes

**Módulo Contabilidad → Otros Fletes**

Se implementó el módulo completo de otros fletes, cubriendo la capa de datos, lógica de negocio, API y la interfaz de usuario.

**Backend (servidor):**
- **Schema MongoDB `schemaOtrosFletes`:** Nueva colección dedicada para registrar fletes adicionales fuera del esquema de fletes estándar.
- **Clase `OtrosFletes`:** Nueva clase de negocio con la lógica de creación, consulta y modificación de registros de otros fletes.
- **Endpoints en API Contabilidad:** Rutas REST para consultar, crear y modificar registros de otros fletes, con autenticación de usuario.
- **Eventos WebSocket contabilidad:** Nuevos eventos de socket para sincronización en tiempo real de otros fletes entre clientes.
- **Actualización `init.js`:** Se registró la nueva colección en la inicialización de MongoDB.
- **Actualización `schemaLotes`:** Ajuste en el esquema de lotes relacionado con la integración.

**Agrupación de fletes (PRs #46 y #47):**
- **Refactor de agrupación en `Contabilidad.js`:** Dos ciclos de ajuste que reestructuraron la lógica de agrupación de fletes (238 líneas agregadas, 104 eliminadas), permitiendo consolidar correctamente los distintos tipos de flete en los informes y documentos.

**Frontend (desktop):**
- **Vista `OtrosInformesContabilidad`:** Nueva vista con tabla, filtros (`FiltrosOtrosFletes`) y formulario de registro (`RegistroOtrosFletes`) para gestionar otros fletes.
- **Refactor de `InformeFletesContabilidad`:** Reestructuración mayor del componente de informe de fletes de contabilidad, consolidando la visualización de fletes normales y otros fletes en una misma vista coherente.

### 4. Contratos de Personal (Nuevo)

**Módulo Talento Humano → Gestión Laboral → Contratos**

- **Vista `ContratosPersonal`:** Nueva sección en gestión laboral para visualizar y administrar los contratos del personal de planta.
- **Modal `ModalContratosPersonal`:** Modal dedicado para crear, consultar y gestionar contratos individuales de personal, con campos de fecha, tipo de contrato y datos del trabajador.
- **Registro en `componentRegistry`:** El módulo fue registrado en el sistema de navegación de la app desktop para acceso desde el menú principal.

### 5. Módulo de Personal – Mejoras

**Módulo Personal → Higiene / Ingreso**

- **Campo apellido en personal:** Se agregó el campo apellido al modelo de personal para completar la información del trabajador.
- **Validaciones Zod en personal:** Se incorporaron validaciones Zod en los esquemas de personal para mayor integridad de datos.
- **Refactor higiene personal:** Se reestructuró el módulo de higiene de personal, simplificando el flujo de carga de documentos y mejorando los estilos visuales.
- **Corrección estilos higiene personal:** Mejoras de estilos en la vista de higiene personal de la app desktop.

---

### 6. Rendimiento del Servidor

- **Corrección latencia P95 elevada:** Se identificó y corrigió la causa de latencia elevada en requests HTTP, eliminando además logs ruidosos que saturaban la salida.
- **Optimización del BrowserPool:** Se mejoró la forma en que los worker threads gestionan el pool de browsers para generación de PDFs, reduciendo consumo de recursos.
- **Índice para colección de insumos:** Se creó un índice en MongoDB para la colección de insumos, mejorando los tiempos de consulta.
- **Mejoras de rendimiento generales en app:** Cambios en la app desktop para mejorar el rendimiento de renderizado y carga.

---

### 7. Automatización del Despliegue (CI/CD)

- **Migración a GitHub Actions:** Se implementó un pipeline de despliegue automatizado usando GitHub Actions, reemplazando el proceso manual anterior.
- **Deploy continuo:** El servidor ahora se despliega automáticamente al hacer push a la rama principal, reduciendo el tiempo y riesgo del proceso de actualización.

---

### 8. Restricción de Cierre de Contenedores

**Módulo Proceso / Logística → Contenedores**

- **Bloqueo de cierre hasta completar cuarto frío:** Se implementó una restricción que impide cerrar un contenedor mientras existan elementos pendientes de ingreso a cuarto frío. El sistema valida el estado de todos los elementos asociados al contenedor antes de permitir la operación de cierre, garantizando la trazabilidad y el cumplimiento del protocolo de frío en cada despacho.

---

### 9. Accesos Parciales a la Colección de Cargos

**Módulo RBAC → Permisos**

- **Accesos parciales en colección `cargos`:** Se incorporaron permisos granulares sobre la colección de cargos, permitiendo asignar a roles específicos acceso de lectura, creación o modificación de forma independiente. Esto amplía el modelo RBAC del sistema para controlar con mayor precisión quién puede consultar o administrar la estructura de cargos del personal.

---

### 10. Scripts de Análisis de Datos

- **Script info exportación y descarte:** Se creó un script para obtener información consolidada de la exportación y el descarte, facilitando el análisis offline.
- **Script kilos calidad por lote:** Nuevo script para extraer y analizar los kilos de calidad agrupados por lote.
- **Script carnet:** Se creó el script de apoyo para generar carnets desde consola.
